### Capture Endpoint (`/capture` POST)

Fire-and-forget form capture. Validates email syntax, bounds-checks quizNodeId 1‚Äì13, dispatches to KV namespace (`SUBMISSIONS`) and GHL webhook (`GHL_WEBHOOK_URL`) in parallel via `Promise.all`. Any sink failure is logged but never blocks the 200 response. In local dev, KV is unbound ‚Äî degrades gracefully with a warning.

## Gotchas

- **Dev server binds 0.0.0.0** ‚Äî required for tunnel access. `allowedHosts` in `astro.config.mjs` has explicit allowlist.
- **Nodes 6‚Äì13 are locked** ‚Äî `status: 'locked'` in `hub.config.ts`. Do not change without client approval.
- **Node 5 ‚Üí `/water/`** ‚Äî Root Healing node routes to the hydration funnel, not `/nodes/5`.
- **Vessel mode / nav variant** ‚Äî querystring overrides available (`?vessel=visible`, `?nav=spiral-first`) for A/B testing. Defaults in `hub.config.ts` `ui` object.
- **Functions vs API routes** ‚Äî `functions/` directory is legacy. Use `src/pages/*.ts` with `export const prerender = false` for server routes.
- **No `.env` committed** ‚Äî env vars: `KEYSTATIC_STORAGE_KIND`, `GHL_WEBHOOK_URL`, `CLOUDFLARE_API_TOKEN`.
- **Empty affiliate URLs** ‚Äî `Anespa DX` and `K8 Kangen` have `affiliateUrl: ''` in `hydration.config.ts` ‚Äî these are client-gated (tracked in GH#49).
- **`ghl.quizFormUrl` is empty** ‚Äî local fallback to `/quiz`; affiliate routing tracked in GH#49 and GH#56.
- **No lockfile committed** ‚Äî `package-lock.json` exists but should be regenerated; use `npm ci` in CI, `npm install` locally.
