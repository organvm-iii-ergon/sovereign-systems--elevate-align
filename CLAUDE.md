## Configuration Layering

The codebase separates **immutable identity** from **mutable surface** through a layered structure in `src/data/`. Each layer modulates rendering downstream; edits to one layer can have invisible knock-on effects in others.

1. **`hub.config.ts`** â€” master metadata; the single source of truth for node count (13), pillar definitions (4), branch list (6), domain/GHL URLs, and UI defaults.
2. **`sacred-geometry-primitives.ts`** â€” immutable geometric essence per `EnvVar` (PYR, OCULUS, DYAD, PYRAMIS, HYDOR, MANDORLA, KENOSIS, SHATKONA, PADMA, BODHI, TETRAD, OKTAEDRON, ANKH); vertex counts, PHI exponents, symmetry types.
3. **`icon-worlds.ts`** â€” per-node visual physics (particles, palette, gravity, behavior); one `IconWorld` per node.
4. **`lens-geometry.ts`** â€” tradition-specific transformations (Egyptian, Sanskrit-Vedic, Greek-classical, Christian-mystical, Jungian, physics-elemental, modern-wellness); how essences appear through each lens.
5. **`naming-chains.ts`** â€” bindings (`EnvVar` Ă— `Lens` â†’ surface name); `chainsFor(envVar)` returns chronological lineage, `viewThroughLens(lens)` returns all 13 nodes through one tradition.

**When changing node identity:** check all 5 layers. Adding a `QuizTheme` tag, for example, requires both `hub.config.ts` (node theme list) AND `src/pages/quiz.astro` (button labels must match exactly).

## Capture Pipeline

**Endpoint:** `POST /capture` â€” `src/pages/capture.ts` (Astro APIRoute; replaces prior `functions/capture.ts` because the Cloudflare adapter's `_worker.js` takes precedence over `functions/`).

**Payload:** `CapturePayload` â€” `email` (required), `name`, `source`, plus quiz extension fields (`quizNodeId` 1..13, `quizScore` 0..100, `quizPath`, `selectedPillar`, `selectedPhase`).

**Sinks (additive, isolated):**
1. **KV** (`SUBMISSIONS` namespace) â€” full payload + `ipHint` (last octet only, from `CF-Connecting-IP`). Optional; degrades gracefully if KV not bound.
2. **GHL webhook** (`GHL_WEBHOOK_URL` env var) â€” fire-and-forget POST with flat JSON envelope. Optional; skips silently if env var unset.
3. **Extension points** in place for D1, email, etc. â€” additive Promise.all branches.

**Contract:** Always returns 200 `{success: true}` for valid email. Quiz UX shows result *before* the capture network call completes â€” this is a UX invariant, not a bug. Sink failures never block the response.

**Adding a sink:** new Promise.all branch; no contract change. **De-identification:** only the last octet (or last `::` segment for IPv6) of the IP is kept.
